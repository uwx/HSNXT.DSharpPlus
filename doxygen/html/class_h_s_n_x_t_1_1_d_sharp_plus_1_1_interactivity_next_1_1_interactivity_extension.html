)
inline

Creates a reaction-based poll. Resolves to a ReactionCollectionContext containing the relevant emote reactions.

Parameters
messageThe message to run the poll on
emojiSourceThe poll selection options. Each emote will be added to the message as a reaction. Default are thumbs up and thumbs down emojis.
timeoutTimeout for the waiting period. If not specified, defaults to InteractivityConfiguration.Timeout.
ctCancellation token that can be used to end the waiting period early.
Returns
Task that resolves to a ReactionCollectionContext.
Exceptions
ArgumentNullExceptionIf message is null
InvalidOperationExceptionIf emojiSource is empty

Sends a paginated message. A paginated message is a message that can be controlled and updated dynamically by clicking on certain reactions.

Parameters
channelChannel to send the paginated message to
userThe user that is allowed to interact with the paginated message
messagePagesPages for this message, you can create your own Page instances or use either GeneratePagesInEmbeds (for embeds) or GeneratePagesInStrings (for text-based messages)
ctCancellation token that can be used to end the pagination.
timeoutTimeout until pagination ends. If not specified, defaults to InteractivityConfiguration.Timeout.
finishBehaviourOverrideAction to perform on the message once the pagination ends. Defaults to InteractivityConfiguration.PaginationBehavior
emojisThe emojis to use for pagination controls. Defaults to InteractivityConfiguration.DefaultPaginationEmojis
Returns
A task that resolves once the pagination process finishes
Exceptions
ArgumentNullExceptionIf channel , user or messagePages is null

◆ GeneratePagesInEmbeds()

IEnumerable<Page> HSNXT.DSharpPlus.InteractivityNext.InteractivityExtension.GeneratePagesInEmbeds ( string  input,
DiscordEmbed  source = null 
)
inline

Turns a long string into an enumerable containing pages holding parts of the input string, split at around the Discord length limit, placed in embeds.

Parameters
inputThe string to paginate
sourceAn optional embed to use as the base for forming each page embed. By default, an empty embed is used. Usually, if provided, the embed's title and description will be replaced by the page header and contents, respectively, but which components get replaced is implementation-defined and may change in future (breaking) updates.
Returns
An enumerable of pages, can be passed to SendPaginatedMessage
Exceptions
ArgumentExceptionIf input is null or empty
